🧹 Cleaning operations applied

These transformations run automatically on every file import.

Full file parsed end-to-end — no early cutoff on repeated voucher sections
Multi-row credit notes merged — each voucher becomes flat item-level rows
Rate read from detail sub-rows only (never from blank main row)
Gross = Qty × Rate computed if Gross is missing or zero
Gross validated: rows where Gross ≠ Qty × Rate (±₹1 tolerance) are flagged
Gross allocated across multi-item vouchers proportionally by value
Duplicate detection: Credit Note + Item + Qty + Rate composite key
January re-export duplicates removed: vouchers in Jan AND a later month → Jan rows deleted
Jan repeat detection counts unique voucher numbers (not rows) — multi-item CNs never falsely flagged
Cancelled vouchers excluded automatically
Grand Total rows removed from output
Customer name: longer of Particulars vs Buyer column used
_x000D_ carriage-return artifacts stripped from narration text
All spaces trimmed from all text fields
Product names parsed: Clean Name, Part Number, Make, Tracker extracted
Sorted by Date ascending
Missing Qty/Rate default to 0 and appear in Error Log
SEZ/export items (0 GST): gross falls back to value — flagged as info
